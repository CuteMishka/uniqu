# UNIQU CRM

Лендинг, CRM для заявок и редактор контента работают на одном Node.js-сервере без внешних зависимостей.

## Запуск

```powershell
npm start
```

После запуска:

- лендинг: `http://127.0.0.1:5173/`
- CRM: `http://127.0.0.1:5173/admin.html`
- проверка API: `http://127.0.0.1:5173/api/health`

## Защита CRM паролем

Перед публикацией обязательно задайте пароль администратора:

```powershell
$env:ADMIN_USER="admin"
$env:ADMIN_PASSWORD="используйте-длинный-уникальный-пароль"
npm start
```

Без `ADMIN_PASSWORD` авторизация отключена, что удобно только для локальной разработки.

## Контакты Узбекистана и аналитика

Поля можно заполнить в редакторе контента CRM или передать через переменные окружения:

```powershell
$env:UZ_PHONE="+998 ..."
$env:UZ_PHONE_LINK="998..."
$env:TELEGRAM_URL="https://t.me/..."
$env:CLARITY_PROJECT_ID="идентификатор-проекта"
```

После указания `TELEGRAM_URL` кнопка Telegram автоматически появится рядом с WhatsApp и в футере. `CLARITY_PROJECT_ID` включает официальный счётчик Microsoft Clarity без дополнительной правки файлов.

## Данные

- заявки: `data/leads.json`
- контент лендинга: `data/site-content.json`

Запись выполняется атомарно через временный файл. Для размещения в облаке нужен постоянный диск. Если приложение будет запущено в нескольких экземплярах, JSON-хранилище следует заменить общей базой данных.

Текущая конфигурация рассчитана на один постоянно работающий Node.js-процесс. Перед рабочим Vercel-деплоем подключите внешнюю базу данных: файловая система Vercel Functions не подходит для постоянного хранения заявок.

## Возможности CRM

- дашборд, недельная динамика и воронка;
- канбан со сменой этапа перетаскиванием;
- компактная таблица для мобильных устройств;
- поиск и фильтры по статусу, программе и менеджеру;
- карточка клиента, приоритет, ответственный, бюджет и следующий контакт;
- заметки и история действий;
- экспорт отфильтрованных заявок в CSV;
- создание, обновление и удаление заявок;
- многоязычный редактор контента лендинга;
- Basic Auth для админки и ограничение частоты публичных заявок.
